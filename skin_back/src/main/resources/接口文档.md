# MyPetStore接口文档

## 0.数据库及POJO

> 写在前面：
>
> 1. POJO全部使用驼峰命名，数据库使用下划线命名。
>
>    例如：POJO中叫createdAt，数据库中的列叫created_at
>
> 2. **任何主键全部叫id，里面的id不需要大写**
>
>    例如：signon的主键就是userid不是userId，更不是userID。
>
> 3. 主键一定是整数类型，如果需要放字符串单独一列
>
>    例如：itemid只能是整数，像EST-1这种字符串给item_name

### 0.1用户相关

#### 0.1.1signon

##### columns

| 属性           | 类型         | 限制                                                  | 备注                               |
| -------------- | ------------ | ----------------------------------------------------- | ---------------------------------- |
| **userid**     | INT          | NOT NULL，AUTO_INCREMENT                              |                                    |
| **username**   | VARCHAR(20)  | NOT NULL，6-20                                        |                                    |
| **password**   | VARCHAR(255) | NOT NULL，6-20                                        |                                    |
| **email**      | VARCHAR(255) | NOT NULL                                              |                                    |
| **role**       | VARCHAR(20)  | NOT NULL，6-20                                        |                                    |
| **created_at** | TIMESTAMP    | DEFAULT CURRENT_TIMESTAMP                             | 默认当前时间                       |
| **updated_at** | TIMESTAMP    | DEFAULT CURRENT_TIMESTAMP,ON UPDATE CURRENT_TIMESTAMP | 默认当前时间，修改时更改为当前时间 |

##### keys

```sql
PRIMARY KEY (userid)
UNIQUE KEY (email)
```

##### signon.sql

```sql
create table signon
(
    username   varchar(20)                         not null,
    password   varchar(255)                        not null,
    userid     int auto_increment
        primary key,
    created_at timestamp default CURRENT_TIMESTAMP null,
    updated_at timestamp default CURRENT_TIMESTAMP null on update CURRENT_TIMESTAMP,
    role       varchar(20)                         not null,
    email      varchar(255)                        not null,
    constraint signon_pk
        unique (email)
)
    row_format = DYNAMIC;


```

## 1. 用户相关接口

> 支持用户登录、注册、修改密码、获取当前登录用户的注册信息
>
> 管理员可以获取全部客户的信息、重置客户的密码
>
> 客户可以获取自己的信息、修改自己的信息

### 1.1全局接口

#### 1.1.1 注册

##### 1.1.1.1 基本信息

> 请求路径：/account/register
>
> 请求方式：POST
>
> 接口描述：该接口用于注册新用户

##### 1.1.1.2 请求参数

请求参数格式：x-www-form-urlencoded

请求参数说明：

| 参数名称 | 说明     | 类型   | 是否必须 | 备注                        |
| -------- | -------- | ------ | -------- | --------------------------- |
| username | 用户名   | string | 是       | 5~16位非空字符              |
| password | 密码     | string | 是       | 5~16位非空字符              |
| role     | 用户身份 | stirng | 是       | 只能是"admin"或者"customer" |

请求数据样例：

```shell
username:abcdefg
password:abcdefg
role:customer
```

##### 1.1.1.3 响应数据

响应数据类型：application/json

响应参数说明：

| 名称    | 类型   | 是否必须 | 默认值 | 备注                  | 其他信息               |
| ------- | ------ | -------- | ------ | --------------------- | ---------------------- |
| code    | number | 必须     |        | 响应码, 0-成功,1-失败 |                        |
| message | string | 非必须   |        | 提示信息              | 可能返回"用户名已存在" |
| data    | string | 必须的   |        | 返回的数据,jwt令牌    |                        |

响应数据样例：

```json
{
    "code": 0,
    "message": "操作成功",
```

##### 1.1.2.1 基本信息

> 请求路径：/account/login
>
> 请求方式：POST
>
> 接口描述：该接口用于登录

##### 1.1.2.2 请求参数

请求参数格式：x-www-form-urlencoded

请求参数说明：

| 参数名称 | 说明     | 类型   | 是否必须 | 备注                        |
| :------- | -------- | ------ | -------- | --------------------------- |
| username | 用户名   | string | 是       | 5~16位非空字符              |
| password | 密码     | string | 是       | 5~16位非空字符              |
| role     | 用户身份 | stirng | 是       | 只能是"admin"或者"customer" |

请求数据样例：

```shell
username:abcdefg
```

##### 1.1.2.3 响应数据

响应数据类型：application/json

响应参数说明：

| 名称    | 类型   | 是否必须 | 默认值 | 备注                  | 其他信息                                      |
| ------- | ------ | -------- | ------ | --------------------- | --------------------------------------------- |
| code    | number | 必须     |        | 响应码, 0-成功,1-失败 |                                               |
| message | string | 非必须   |        | 提示信息              | 可能返回""用户名不存在"或者"用户名或密码错误" |
| data    | string | 必须     |        | 返回的数据,jwt令牌    |                                               |

响应数据样例：

```json
{

    "message": "操作成功",

```

##### 1.1.2.4 备注说明

> 用户登录成功后，系统会自动下发JWT令牌，然后在后续的每次请求中，浏览器都需要在请求头header中携带到服务端，请求头的名称为 Authorization，值为 登录时下发的JWT令牌。
>
> 注册登录功能要先启动Redis服务
>
> 如果检测到用户未登录，则http响应状态码为401
